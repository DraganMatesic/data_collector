# WatchService — File System Monitoring

**Work Package:** WP-11 | **Milestone:** v0.5.0 | **Status:** IN DEVELOPMENT

## Overview

The WatchService monitors directories for new files and writes events to the `Events` table. It is an **event producer** — it does not dispatch to Dramatiq directly. The TaskDispatcher ([17.1. dramatiq.md](17.1.%20dramatiq.md)) polls the Events table and handles Dramatiq dispatch.

```
WatchService                    Events Table              TaskDispatcher
(file system monitoring)        (database)                (Dramatiq dispatch)
      │                              │                          │
      ▼                              ▼                          ▼
  Detect file ──► Write event ──► Unprocessed ──► Dispatch to Dramatiq
  creation        to Events       event row        via RabbitMQ
```

**Dependencies:** `watchdog`

Developed and tested primarily on Windows, but fully deployable on Linux.

## watchdog Library

The framework uses the `watchdog` library for file system monitoring. watchdog wraps native OS APIs for near-instant event delivery:

| Platform | Observer Class | Underlying OS API | Notes |
|----------|---------------|-------------------|-------|
| Windows | `WindowsApiObserver` | `ReadDirectoryChangesW` | I/O completion ports. Vista+. Requires `pywin32` |
| Linux | `InotifyObserver` | `inotify(7)` | Kernel 2.6.13+. Default 8,192 watch limit per user |
| macOS | `FSEventsObserver` | Apple FSEvents | `KqueueObserver` as fallback |
| All | `PollingObserver` | `os.stat()` polling | Fallback for Docker volumes, network drives |

## EventHandler (Abstract)

Clean separation -- WatchService handles file system complexity, application code implements the 3-method streaming lifecycle:

```python
from abc import ABC, abstractmethod
from datetime import datetime

class EventHandler(ABC):
    """Application-level handler for file system events.

    Supports the streaming lifecycle: events are detected (potentially
    unstable), then stabilized once the file finishes writing.
    """

    @abstractmethod
    def on_event_detected(self, event_data: EventData) -> int:
        """Create an initial event record in the Events table.

        Returns:
            The database ID of the created event row.
        """

    @abstractmethod
    def on_event_stabilized(self, event_id: int, file_size: int | None) -> None:
        """Mark an event as stable with final metadata."""

    @abstractmethod
    def get_last_stable_event(self, path_hash: str) -> datetime | None:
        """Return the stabilized_date for the most recent stable event.

        Returns:
            The stabilized_date timestamp, or None if no stable event exists.
        """
```

The default `IngestEventHandler` writes events to the `Events` table:

```python
class IngestEventHandler(EventHandler):
    """Writes file events to the Events table for TaskDispatcher pickup."""

    def __init__(self, database: Database) -> None:
        self.database = database

    def on_event_detected(self, event_data: EventData) -> int:
        """Create an Events row. Sets stabilized_date if stable, None if streaming."""
        now = datetime.now(UTC)
        with self.database.create_session() as session:
            event = Events(
                app_path=event_data.root_data.app_path,
                file_path=event_data.abs_path,
                event_type=event_data.event_type,
                path_hash=event_data.path_hash,
                country=event_data.root_data.country,
                watch_group=event_data.root_data.watch_group,
                file_size=event_data.file_size,
                stable=event_data.stable,
                stabilized_date=now if event_data.stable else None,
            )
            session.add(event)
            session.commit()
            return event.id

    def on_event_stabilized(self, event_id: int, file_size: int | None) -> None:
        """Update an Events row to mark it as stable."""
        now = datetime.now(UTC)
        with self.database.create_session() as session:
            event = session.get(Events, event_id)
            if event is None:
                return
            event.stable = True
            event.stabilized_date = now
            event.file_size = file_size
            session.commit()

    def get_last_stable_event(self, path_hash: str) -> datetime | None:
        """Query the most recent stabilized_date for a given path_hash."""
        with self.database.create_session() as session:
            statement = (
                select(Events.stabilized_date)
                .where(
                    Events.path_hash == path_hash,
                    Events.stable == True,
                    Events.archive.is_(None),
                )
                .order_by(Events.stabilized_date.desc())
                .limit(1)
            )
            return session.execute(statement).scalar_one_or_none()
```

## WatchService

Multi-root, thread-safe file watcher with production-hardened features:

```python
from watchdog.events import FileSystemEventHandler
from pathlib import Path
import threading

class WatchdogAdapter(FileSystemEventHandler):
    """Stateless adapter -- forwards CREATED and MODIFIED to WatchService."""

    def __init__(self, watch_service: "WatchService"):
        self.watch_service = watch_service

    def on_created(self, event):
        if not event.is_directory:
            self.watch_service.handle_path(event.src_path, EventType.CREATED)

    def on_modified(self, event):
        if not event.is_directory:
            self.watch_service.handle_path(event.src_path, EventType.MODIFIED)


class WatchService:
    """Multi-root directory watcher with streaming-aware event production."""

    def __init__(
        self,
        roots: list[Root],
        event_handler: EventHandler,
        settings: WatchServiceSettings | None = None,
        database: Database | None = None,
    ) -> None:
        resolved_settings = settings or WatchServiceSettings()
        self._roots: dict[str, Root] = {}
        for root in roots:
            normalized_root = self._normalize_path(root.root_path)
            self._roots[normalized_root] = root

        self._event_handler = event_handler
        self._database = database
        self._pending_stability: dict[str, dict] = {}
        self._pending_lock = threading.Lock()
        self._write_queue: queue.Queue = queue.Queue()
        self._stop_event = threading.Event()
        self._observer = None
```

### Platform-Aware Observer Selection

```python
def _create_observer(self, observer_type: str = "auto"):
    """Select observer based on platform with graceful fallback."""
    if observer_type == "polling":
        from watchdog.observers.polling import PollingObserver
        return PollingObserver(timeout=self.poll_interval)

    if sys.platform == "win32":
        try:
            from watchdog.observers import Observer  # WindowsApiObserver
            return Observer()
        except ImportError:
            logger.warning("pywin32 not installed, falling back to PollingObserver")
            from watchdog.observers.polling import PollingObserver
            return PollingObserver(timeout=self.poll_interval)
    else:
        try:
            from watchdog.observers import Observer  # InotifyObserver on Linux
            return Observer()
        except Exception:
            from watchdog.observers.polling import PollingObserver
            return PollingObserver(timeout=self.poll_interval)
```

### File Stability Debounce

Prevents processing half-written files by checking file size twice with a delay:

```python
def _is_stable(self, file_path: str) -> bool:
    """Check if file has finished writing (size stable between two checks)."""
    try:
        s1 = os.path.getsize(file_path)
        time.sleep(self.debounce / 2)
        s2 = os.path.getsize(file_path)
        return s1 == s2 and s1 > 0
    except OSError:
        return False
```

Files that aren't stable on first check are deferred to a background stability monitor thread that re-checks periodically with a configurable timeout (default 60s):

```python
def _monitor_stability(self):
    """Background thread -- re-checks deferred files until stable or timed out."""
    while not self._stop_event.is_set():
        with self._pending_lock:
            now = time.time()
            expired = []
            for path, info in self._pending_stability.items():
                if now - info["first_seen"] > self.stability_timeout:
                    expired.append(path)
                    logger.warning("Stability timeout", path=path)
                elif self._is_stable(path):
                    self._process_event(path, info["root"], info["event_type"])
                    expired.append(path)
            for path in expired:
                del self._pending_stability[path]
        self._stop_event.wait(timeout=self.debounce)
```

### UNC Path Normalization

Handles Windows enterprise network paths (`\\server\share`), forward/backward slashes, and long paths (>248 chars):

```python
def _normalize_path(self, path: str) -> str:
    """Normalize path for consistent hashing and dedup."""
    path = path.replace("/", "\\") if sys.platform == "win32" else path
    if sys.platform == "win32" and path.startswith("\\\\"):
        # UNC path normalization
        path = "\\\\?" + path[1:]  if len(path) > 248 else path
    return os.path.normpath(path)
```

### Event Processing Flow

```python
def handle_path(self, file_path: str, event_type: EventType):
    """Main entry point -- called by WatchdogAdapter for each FS event."""
    file_path = self._normalize_path(file_path)

    # Find which root this path belongs to
    root = self._find_root(file_path)
    if root is None:
        return

    # Extension filtering
    if root.extensions and not self._is_allowed(file_path, root.extensions):
        return

    path_hash = make_hash(file_path)

    if event_type == EventType.CREATED:
        # Add to pending stability, enqueue DetectCommand with stable=False
        with self._pending_lock:
            if file_path in self._pending_stability:
                return  # Absorb duplicate CREATED while streaming
            self._pending_stability[file_path] = {
                "root": root, "event_type": event_type,
                "event_id": None, "first_seen": time.monotonic()
            }
        event_data = EventData(
            abs_path=file_path, root_data=root, event_type=event_type,
            path_hash=path_hash, stable=False,
        )
        self._write_queue.put(DetectCommand(event_data=event_data, normalized_path=file_path))

    elif event_type == EventType.MODIFIED:
        # Absorb MODIFIED during streaming, otherwise check grace period
        with self._pending_lock:
            if file_path in self._pending_stability:
                return  # Absorb MODIFIED during streaming
        file_modification_time = os.path.getmtime(file_path)
        event_data = EventData(
            abs_path=file_path, root_data=root, event_type=event_type,
            path_hash=path_hash, stable=True,
        )
        self._write_queue.put(
            ModifiedCheckCommand(event_data=event_data, file_modification_time=file_modification_time)
        )
```

## Production Mitigations

Watchdog events are **signals, not guarantees**. The framework treats them as hints and uses database-level idempotency (via `EventProcessingStatus`) to handle all edge cases safely:

| Scenario | Issue | Mitigation |
|----------|-------|------------|
| Windows high throughput | `ReadDirectoryChangesW` buffer overflow -- events silently dropped | Periodic reconciliation scan (configurable interval, default 60s) re-scans directories and dispatches any files missing from `Events` table |
| Docker (Linux container + Windows host volume) | `inotify` receives no events for host-side changes | Auto-fallback to `PollingObserver` via `watcher_observer=polling` setting or `DC_WATCHER_OBSERVER=polling` env var |
| Linux production | Default 8,192 inotify watch limit | Set `fs.inotify.max_user_watches=524288` in `/etc/sysctl.conf` or Docker base image |
| Network drives (CIFS/SMB) | Native observers may not receive events | Use `PollingObserver` explicitly; implement reconnection in watcher service |
| Half-written files | Large files (PDF, ZIP) arrive over network -- partial read | File stability debounce: size-check-twice with configurable delay |
| Streaming echoes | MODIFIED events fired immediately after file stabilization | Streaming-aware lifecycle -- MODIFIED absorbed during streaming, grace period (`stabilization_grace`, default 5s) prevents false events after stabilization |
| All environments | Events are signals, not guarantees | DB-level dispatch tracking -- `EventProcessingStatus` prevents double Dramatiq dispatch regardless of how many times the event is written |

## Reconciliation Scanner

A background thread periodically scans all watched directories and writes events for any files not yet tracked in the `Events` table. This catches files missed due to buffer overflow, watcher restart, or manual file placement:

```python
def reconcile(self):
    """Periodic full-directory scan to catch missed events."""
    for root in self.roots.values():
        root_path = Path(root.root_path)
        for file_path in root_path.rglob("*") if root.recursive else root_path.iterdir():
            if not file_path.is_file():
                continue
            if root.extensions and file_path.suffix.lower() not in root.extensions:
                continue

            path_hash = make_hash(str(file_path))
            # Check if already in Events table
            with self.database.create_session() as session:
                exists = session.execute(
                    select(func.count()).select_from(Events)
                    .where(Events.path_hash == path_hash)
                ).scalar()
                if exists == 0:
                    self.handle_path(str(file_path), EventType.CREATED)
```

## Root Loading and Hot-Reload

Roots are loaded from the `watch_roots` table via `load_roots_from_database()`:

```python
from data_collector.messaging.watchservice import load_roots_from_database

roots = load_roots_from_database(database)
```

The function queries active, non-archived rows and parses the JSON `extensions` column into `list[str]`.

At runtime, the reconciliation loop periodically calls `_refresh_roots()` which reloads the database state and compares it to the in-memory root map. Changes are detected automatically:

- **New roots** -- scheduled on the watchdog observer immediately.
- **Removed roots** (archived or deactivated) -- unscheduled from the observer.
- **Changed roots** (same path, different config) -- unscheduled and re-scheduled with the new configuration.

This provides hot-reload of watch roots without restarting the service.

## Manager Integration

WatchService and TaskDispatcher start as daemon threads inside the Manager process:

- **Auto-detect:** The Manager starts WatchService if active WatchRoots exist in the database, and starts TaskDispatcher if a DramatiqBroker is available. No `DC_MANAGER_WATCHER_ENABLED` flags are needed.
- **Shutdown order:** TaskDispatcher stops first (drains its dispatch queue), then WatchService stops (drains the write queue). This ensures no events are dispatched for files that haven't been fully written.
- **Independent operation:** WatchService and TaskDispatcher can run independently. WatchService writes events regardless of whether TaskDispatcher is running. TaskDispatcher polls events regardless of whether WatchService produced them.

## Architectural Decisions

### 1. Streaming-aware lifecycle (no TTL-based dedup)

TTL-based deduplication is unreliable for large files over slow networks. Instead, the system tracks streaming state in the database: CREATED events start with `stable=False`, a background stability monitor checks file size stability, and updates the Events row to `stable=True` with `stabilized_date` when the file finishes writing. MODIFIED events during streaming are absorbed (coalesced into the original CREATED event). Legitimate re-modifications only produce new events when `file_mtime > stabilized_date + grace_seconds`.

### 2. No DELETE events

WatchdogAdapter only forwards CREATED and MODIFIED. Downstream actors (OCR, extraction) need file content -- DELETE events point to non-existent files and cause downstream errors. `EventType` enum retains `DELETED=3` for forward compatibility.

### 3. Non-blocking design with queue-based writer

The watchdog thread must never be blocked by DB operations or stability checks. Four daemon threads: watchdog observer (OS events), writer (drains a `queue.Queue` for all DB I/O), stability monitor (deferred file checks), and reconciliation scanner (catch-all safety net). On graceful shutdown, the writer thread drains the queue before exiting. On ungraceful shutdown, the reconciliation scanner catches missed files on next startup.

### 4. MODIFIED grace period

Uses `file_mtime > stabilized_date + grace_seconds` (default 5s) to prevent false MODIFIED events from streaming echoes or filesystem timestamp jitter immediately after stabilization.

### 5. Corrupted file handling

WatchService does not validate file integrity. Downstream workers detect corruption, mark `Events.corrupted=True`, and return cleanly (no Dramatiq retry). If the file is replaced, a new MODIFIED event triggers reprocessing.

### 6. Worker read-only convention

Source files in watched directories are read-only inputs. Workers read freely (pdfplumber, pytesseract, pdf2image all open in read mode without modifying the source). Workers write output to separate directories, never back to the watched directory. Reading does not trigger watchdog MODIFIED events (inotify `IN_MODIFY` and Windows `FILE_NOTIFY_CHANGE_LAST_WRITE` only fire on content writes).

### 7. Hot folder convention

Watched directories are hot folders -- temporary holding areas. Workers move processed files to permanent storage (`Events.destination_path`) and delete the source from the hot folder. If storage is unavailable, the file stays in the hot folder and Dramatiq retries. After max retries, the message goes to the dead letter queue.

### 8. Database outage handling

Writer thread detects DB failure and enters exponential backoff (1s to 30s cap). Events accumulate in the unbounded in-memory queue. When DB recovers, the writer drains the backlog. If the process crashes during a DB outage, reconciliation catches up after restart.

### 9. Startup recovery

On `start()`, WatchService queries `Events WHERE stable=False AND archive IS NULL` and re-adds entries to `_pending_stability`. Then runs a full reconciliation scan to catch files added during downtime.

### 10. Events table concurrency

No deadlock risk: single writer thread serializes all writes, operations are temporally sequential (WatchService writes, TaskDispatcher reads, workers update), PostgreSQL MVCC ensures readers never block writers, and all transactions are short (commit per operation).

## Hot folder source types

| Source | How it works | Observer |
|--------|-------------|----------|
| Local directory | Direct monitoring | Native (auto) |
| Remote directory (SMB/CIFS/NFS mount) | Mounted as local path | PollingObserver (`DC_WATCHER_OBSERVER=polling`) |
| SFTP | Separate poller app downloads to local hot folder | Native (auto) on the local folder |

WatchService only monitors local filesystems. Remote sources (SFTP, API, scraper) deposit files into the local hot folder via separate components.
