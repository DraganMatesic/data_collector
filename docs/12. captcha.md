# Captcha Solving

**Implementation Status:** IMPLEMENTED | **Work Package:** WP-06 | **Milestone:** v0.3.0

## Overview

Provider-based captcha solving architecture for automated captcha resolution during data collection. Handles reCAPTCHA v2/v3, Cloudflare Turnstile, and image captchas encountered during scraping workflows. All HTTP communication uses the centralized `Request` class ([4.4. request.md](4.4.%20request.md)).

The architecture follows the same provider pattern as proxy management ([15. proxy.md](15.%20proxy.md)): a `BaseCaptchaProvider` ABC defines the solve interface, and concrete implementations (starting with `AntiCaptchaProvider`) handle provider-specific API communication.

## Setup

1. **Create an AntiCaptcha account** at anti-captcha.com
2. **Get your API key** from the AntiCaptcha dashboard
3. **Set the environment variable:**
   ```bash
   export DC_CAPTCHA_API_KEY="your_anticaptcha_api_key"
   ```

## Supported Captcha Types

| Type | Task Type Enum | Solution Method |
|------|---------------|----------------|
| reCAPTCHA v2 | `RECAPTCHA_V2` | Submit sitekey + URL, receive token |
| reCAPTCHA v2 (proxy) | `RECAPTCHA_V2_PROXY` | Same, with proxy fields for the solver worker |
| reCAPTCHA v3 | `RECAPTCHA_V3` | Submit sitekey + URL + action + minScore, receive token |
| Turnstile | `TURNSTILE` | Submit sitekey + URL, receive token |
| Turnstile (proxy) | `TURNSTILE_PROXY` | Same, with proxy fields for the solver worker |
| Image captcha | `IMAGE` | Submit base64-encoded image, receive recognized text |

## Architecture

```
BaseCaptchaProvider (ABC)
    |
    +-- solve_recaptcha_v2(site_key, page_url) -> CaptchaResult
    +-- solve_recaptcha_v2_proxy(..., proxy fields) -> CaptchaResult
    +-- solve_recaptcha_v3(site_key, page_url, action, min_score) -> CaptchaResult
    +-- solve_turnstile(site_key, page_url) -> CaptchaResult
    +-- solve_turnstile_proxy(..., proxy fields) -> CaptchaResult
    +-- solve_image(image_data: bytes, page_url: str) -> CaptchaResult
    +-- get_balance() -> float
    +-- report_correct(task_id, task_type) -> bool
    +-- report_incorrect(task_id, task_type) -> bool
    +-- _create_and_poll()  # Shared retry logic + _persist_log
    |
    +-- AntiCaptchaProvider  (api.anti-captcha.com)
    +-- (Future: TwoCaptchaProvider, CapSolverProvider, CapMonsterProvider)
```

### Retry Flow

```
_create_and_poll(create_fn, task_type, poll_fn)
    |
    for attempt in range(1 + max_retries):
        |
        +-- create_fn() -> (task_id, start_time)
        +-- poll_fn(task_id, task_type, start_time)
        |       |
        |       +-- CaptchaResult -> return (success)
        |       +-- CaptchaTimeout -> retry (if attempts remain)
        |
    raise CaptchaTimeout (all retries exhausted)
```

## Usage

```python
from data_collector.captcha import AntiCaptchaProvider, CaptchaMetrics
from data_collector.settings.captcha import CaptchaSettings
from data_collector.utilities.request import Request

settings = CaptchaSettings()  # Reads DC_CAPTCHA_* env vars
metrics = CaptchaMetrics()

captcha = AntiCaptchaProvider(
    api_key=settings.api_key,
    request=Request(),
    timeout=settings.timeout,
    max_retries=settings.max_retries,
    poll_interval=settings.poll_interval,
    metrics=metrics,
)

# reCAPTCHA v2
result = captcha.solve_recaptcha_v2(
    site_key="6Le-wvkSAAAAAPBM...",
    page_url="https://example.com/form",
)
print(result.solution)  # Token string

# reCAPTCHA v3
result = captcha.solve_recaptcha_v3(
    site_key="6Le-key",
    page_url="https://example.com",
    action="login",
    min_score=0.9,
)

# Turnstile
result = captcha.solve_turnstile(
    site_key="0x4AAAAAAA...",
    page_url="https://cf-protected.com",
)

# Image captcha
result = captcha.solve_image(image_bytes, page_url="https://example.com/captcha")
print(result.solution)  # Recognized text

# Account balance
balance = captcha.get_balance()
```

### Proxy-Enabled Solving

When the target site serves different captcha challenges based on IP, pass proxy fields so the solver worker uses the same IP as the scraper:

```python
result = captcha.solve_recaptcha_v2_proxy(
    site_key="6Le-key",
    page_url="https://example.com",
    proxy_type="http",
    proxy_address="1.2.3.4",
    proxy_port=8080,
    proxy_login="user",
    proxy_password="pass",
)
```

### Integration with BaseScraper

```python
class ProtectedPortal(BaseScraper):
    base_url = "https://portal.example.com"

    def __init__(self, database, **kwargs):
        super().__init__(database, **kwargs)
        settings = CaptchaSettings()
        self.captcha = AntiCaptchaProvider(
            api_key=settings.api_key,
            request=Request(),
            timeout=settings.timeout,
            max_retries=settings.max_retries,
            poll_interval=settings.poll_interval,
        )

    def collect(self):
        for item in self.work_list:
            response = self.request.get(f"{self.base_url}/search/{item.reg_number}")

            site_key = detect_recaptcha(response.content)
            if site_key:
                result = self.captcha.solve_recaptcha_v2(
                    site_key=site_key,
                    page_url=f"{self.base_url}/search/{item.reg_number}",
                )
                response = self.request.post(
                    f"{self.base_url}/search",
                    data={"g-recaptcha-response": result.solution, "query": item.reg_number},
                )

            data = parse_results(response.content)
            self.store(data)
            self.solved += 1
```

## CaptchaMetrics

Thread-safe metrics collector following the `RequestMetrics` pattern. Safe to share across threads in multi-threaded scrapers.

| Counter | Method | Description |
|---------|--------|-------------|
| `submitted` | `record_submitted()` | Tasks submitted to the provider |
| `solved` | `record_solved(cost)` | Tasks successfully solved |
| `timed_out` | `record_timed_out()` | Tasks that exceeded timeout |
| `failed` | `record_failed()` | Tasks that failed with API errors |
| `total_cost` | (accumulated by `record_solved`) | Total USD spent |

```python
metrics = CaptchaMetrics()
# ... use with provider ...
stats = metrics.log_stats(logger)
# {"submitted": 10, "solved": 8, "timed_out": 1, "failed": 1, "total_cost": 0.016, "solve_rate_percent": 80.0}
```

## CaptchaLog Table

Operational log table recording every captcha task submitted to a provider. Does not follow DataTableMixin (no sha, archive, date_modified columns). Follows the `ProxyReservation` pattern.

| Column | Type | Notes |
|--------|------|-------|
| `id` | `BigInteger` (PK) | `auto_increment_column()` |
| `app_id` | `String(64)` | FK to `apps.app`, indexed |
| `runtime` | `String(64)` | FK to `runtime.runtime`, indexed |
| `provider_name` | `String(64)` | e.g. `"anti_captcha"` |
| `task_id` | `String(128)` | Provider's task ID, indexed |
| `task_type` | `String(32)` | `CaptchaTaskType` value |
| `page_url` | `Text` | Origin + path only (query string stripped) |
| `cost` | `Float` | Cost in USD (server default 0) |
| `elapsed_seconds` | `Float` | Seconds elapsed before outcome |
| `status` | `BigInteger` | FK to `c_captcha_solve_status.id` |
| `is_correct` | `Boolean` | NULL=unknown, updated by report methods |
| `date_created` | `DateTime` | `server_default=func.now()` |

Logging is optional. Pass `database`, `app_id`, and `runtime` to the provider constructor to enable persistence. When any of the three is missing, `_persist_log` is a no-op.

## CaptchaLogError Table

Provider error details for failed captcha solve attempts. One-to-one with CaptchaLog, following the CaptchaLog/CaptchaLogError split pattern (`data_collector/tables/captcha.py`). A row is created only when the provider returns an error (status FAILED or TIMED_OUT). Successful solves have no error record.

| Column | Type | Notes |
|--------|------|-------|
| `id` | `BigInteger` (PK) | `auto_increment_column()` |
| `captcha_log_id` | `BigInteger` | FK to `captcha_log.id`, unique, indexed, CASCADE |
| `error_code` | `String(128)` | Provider error code |
| `error_description` | `Text` | Provider error message |
| `error_category` | `BigInteger` | FK to `c_captcha_error_category.id` |
| `date_created` | `DateTime` | `server_default=func.now()` |

**Rejection evidence is out of scope.** `CaptchaLog.is_correct` records whether the target site accepted the solved token but does not store why a token was rejected. Target-site HTTP responses are file artifacts, not database data. Developers who need dispute evidence should persist HTTP responses via Storage (WP-12) or their own pipeline, keyed by `task_id` for correlation. `CaptchaLogError` captures provider-side errors only (error codes from the captcha API), not target-site rejection details.

```python
captcha = AntiCaptchaProvider(
    api_key=settings.api_key,
    request=Request(),
    database=database,
    app_id=app_id,
    runtime=runtime_id,
)
```

## CaptchaErrorCategory

Provider-agnostic error classification. Each provider maps its own error codes to these categories internally. Defined in `data_collector/enums/captcha.py`.

| Category | Value | Scraper Action |
|----------|-------|----------------|
| `AUTH` | `"auth"` | Abort (fatal) -- bad API key, suspended account |
| `BALANCE` | `"balance"` | Abort (fatal) -- zero or negative balance |
| `PROXY` | `"proxy"` | Rotate proxy and retry |
| `TASK` | `"task"` | Abort (code bug) -- unsupported task, bad parameters |
| `SOLVE` | `"solve"` | Retry or skip -- unsolvable captcha, worker failures |
| `RATE_LIMIT` | `"rate_limit"` | Backoff and retry -- no available workers |
| `UNKNOWN` | `"unknown"` | Log and decide |

`CaptchaError` carries a `category` attribute (defaults to `CaptchaErrorCategory.UNKNOWN`). AntiCaptcha maps 30+ error codes to categories via `_ERROR_CATEGORY_MAP`.

```python
from data_collector.captcha.models import CaptchaError

try:
    result = captcha.solve_recaptcha_v2(site_key, page_url)
except CaptchaError as error:
    if error.category in (CaptchaErrorCategory.AUTH, CaptchaErrorCategory.BALANCE):
        raise  # Fatal -- abort the scraper
    if error.category == CaptchaErrorCategory.PROXY:
        rotate_proxy()
        continue  # Retry with new proxy
```

## Reporting

Scrapers call `report_correct()` / `report_incorrect()` after verifying the captcha solution against the target page. Reporting improves solver quality and can trigger refunds for incorrect solves.

```python
# After verifying the solution worked
captcha.report_correct(result.task_id, result.task_type)

# After verifying the solution was wrong
captcha.report_incorrect(result.task_id, result.task_type)
```

Report methods return `True` on success, `False` if reporting is not supported for the given task type or the API call fails. Default implementation on `BaseCaptchaProvider` returns `False`. AntiCaptcha supports:

| Task Type | `report_correct` | `report_incorrect` |
|-----------|------------------|-------------------|
| RECAPTCHA_V2 | `/reportCorrectRecaptcha` | `/reportIncorrectRecaptcha` |
| RECAPTCHA_V2_PROXY | `/reportCorrectRecaptcha` | `/reportIncorrectRecaptcha` |
| RECAPTCHA_V3 | `/reportCorrectRecaptcha` | `/reportIncorrectRecaptcha` |
| TURNSTILE | Not supported | Not supported |
| TURNSTILE_PROXY | Not supported | Not supported |
| IMAGE | Not supported | `/reportIncorrectImageCaptcha` |

When `_log_enabled` is `True`, successful report calls also update the `is_correct` column in `CaptchaLog`.

## Future Provider Compatibility

The architecture supports adding providers by subclassing `BaseCaptchaProvider`. All four major providers use the same createTask/getTaskResult REST pattern:

| Provider | API Compatibility | Notes |
|----------|------------------|-------|
| AntiCaptcha | Reference implementation | Human-powered solvers |
| 2Captcha | Fully compatible | Same API strings |
| CapSolver | Compatible (different casing) | AI-powered, fastest |
| CapMonster Cloud | Fully compatible (AC-compatible API) | AI-powered, cheapest |

Adding a new provider requires only a new file with a `BaseCaptchaProvider` subclass, provider-specific task type mapping, and solution field mapping.

## Security Considerations

**Proxyless methods are recommended by default.** The standard solve methods (`solve_recaptcha_v2()`, `solve_turnstile()`, etc.) do not expose any proxy credentials -- the captcha provider uses their own IP to solve the challenge. Use these unless proxy-enabled solving is specifically required.

**Proxy credential exposure.** When using the `_proxy` variants (`solve_recaptcha_v2_proxy()`, `solve_turnstile_proxy()`), proxy credentials (address, port, login, password) are sent to the captcha provider's REST API. The provider's worker uses these credentials to connect through your proxy during the solve window. This means the third-party provider has full access to your proxy credentials for the duration of the request -- and potentially beyond, since credentials may remain valid after the solve completes.

**Mitigation -- dedicated proxy zone.** If proxy-enabled solving is required (because the target site binds captcha tokens to the visitor's IP), use a separate proxy zone with limited privileges rather than sharing the main scraping proxy credentials. For example, with BrightData, create a dedicated zone for captcha solving with restricted bandwidth or session limits. This limits the blast radius if credentials are compromised or misused by the provider.
