# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 0.x.x   | Yes (current development line) |

Only the latest release on the `master` branch receives security updates.

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

### Process

1. Send an email to 32225602+DraganMatesic@users.noreply.github.com with:
   - A description of the vulnerability
   - Steps to reproduce or a proof of concept
   - The affected version(s)
   - Any potential impact assessment

2. You will receive an acknowledgment within 7 business days.

3. The maintainer will investigate and provide a timeline for a fix. Critical
   vulnerabilities will be prioritized.

4. Once a fix is released, the reporter will be credited in the changelog
   (unless anonymity is requested).

### Scope

This policy covers the `data_collector` framework code in this repository.
Third-party dependencies are outside the scope of this policy; vulnerabilities
in dependencies should be reported to their respective maintainers.

### Disclosure

The maintainer follows coordinated disclosure. Please allow reasonable time for
a fix before any public disclosure.
